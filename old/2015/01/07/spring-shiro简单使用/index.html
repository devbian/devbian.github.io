spring shiro简单使用

想使用shiro做权限控制,只是网络上好多的shiro资料让人好心塞,过于复杂,让刚接触shiro的人很不容易明白.
而对于学习,最好的情况就是在了解一点理论后,就开始demo的编写,从一个个小demo中学习.

对于shiro理论知识,infoq的这篇文章. 一定要耐心一点看完. 主要就是知道Subject, SecurityManager,以及Realm这三个概念.
知道了这些以后,就可以实例的编写了.
别的高大上的东西,在demo运行后,你会自然而然的想去了解.

这里的例子就是用shiro做一个url路径的权限控制.
这里有/home /login /register /user三个地址.
要求前三个地址可以直接访问,而/user必须是登录后的用户才可以访问,否则跳转到登录页面.

对于java web项目,最首先的都是配置文件,这里也不例外

重点在于shiro的配置.
applicationContext-shiro.xml 要配置shiroFilter bean,还有securityManager

每一个shiro项目,至少拥有一个Realm类.
Realm就是用于认证与授权
由于我们这个例子只是简单的url访问控制,所以,只要认证功能就行了,所以,只要复写doGetAuthenticationInfo方法就可以了.

为了保持项目的简单,没有使用数据库,只是使用假数据模拟.
SimpleUser user = new SimpleUser("abc", "123");
当用户使用”abc”,”123” 登录时,就可以直接登入
doGetAuthenticationInfo 方法的最后是返回一个SimpleAuthenticationInfo.
